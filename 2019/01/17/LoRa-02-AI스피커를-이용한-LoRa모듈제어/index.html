👉 LoRa쉴드에 대한 간단한 설명

✋ LoRa쉴드는 어떤 역할이죠 ?



이전 1부포스팅에서 LoRa MAC구조의 설명중 그래서 로라쉴드는 어떤 부분의
기능을 도와주는지 몰랐던 분들을 위해 간략하게 설명하면
기존의 LoRa 하드에어와 LoRa Alliance가 정한 ClassA ~ C 사이의 설정및 기존 초기화하여
노드들과의 연결하는 부분들을 모듈화하여 간단하게 연결, 설정, 기능, 암호화등을 도와주고
아두이노와 연결, 사용가능하게 하는게 코드주 LoRa쉴드의 역할이라고 간단히 생각하시면
됩니다 ~

👉 Gateway 구현

✋ Gateway 역할


1부에 소개되었던 사진입니다.
LoRaWAN이나 LoRa를 사용시 권장하는 방식인데
제가 번호를 태깅해놓았는데 여기에 해당하는 1번 역할을 하는것이 Gateway입니다.
end-device 사이의 메세지를 받고 central network server를 구성해서 보내두되구
다른 gateway끼리 통신을 주고 받아 자유롭게 구성을 하는
즉, 클라이언트가 end-device에 내리기전에 거치는 곳이라고 생각을 하면되는데
AI스피커를 활용해서 LoRa모듈을 제어하기위해서 거치는 플로우는
사용자 명령 -> 외부망 -> LoRa end-device로의과정인데
외부망에서 LoRa end-device로의 직접적인 전달은 주파수가 달라 불가하기때문에
외부망과 LoRa end-device 사이의 LoRa게이트웨이를 설계하기로 계획했습니다.

✋ Gateway 구성



위에 보이는것처럼 게이트웨이는 집안의 외부망(공유기)에 접속하고자
베이스에는 WemosD1 (wifi Module)을 사용하여 공유기에 접속 특정 URL의 Request를
인식하거나, 외부망에서의 원격제어를 위해 사용하였고 그 위에 LoRa쉴드를 올려서
LoRa End-device에 신호를 전달하는 식으로 구성하고자 이렇게 구성하였습니다.

✋ Gateway 구현

이런식으로 Wemos AP를 연결시키고 나서

setup함수에 Wemos 보드 IP + on이나 off의 URL에 트리거를 겁니다.
트리거에 해당하는 함수를 하나 만들어놓아서 LoRa모듈에게 전체 브로드캐스팅하는
msg값을 0 = 기본, 1 = 키는것, 2 = 끄는것 이런식으로 내부적으로 정해서
gateway 부분에서는 보내고, end-device에서의 신호를 받고나서 받았다는 신호를 다시
게이트웨이 에게 보낼때 어떻게 처리해야될지를 중점적으로 생각하시고 구현하시면
좀더 다양한 컨셉에 적용가능 할 것 같습니다.

현재는 msg라는 String하나로만 하였지만 여러 단말 end-device가 있다면
msg값뿐만이아닌 id값이라는 변수를 하나더 만들어서 end-device에서 보내는 msg를
자기 알고리즘에 분류후 id값도 확인해서 쓰면 어떨까 싶다는생각이였습니다.
EX )
msg = 11 이면 1번에게 1(키라는) 명령
msg = 21 이면 2번에게 1(키라는) 명령 이런식으루요 !

👉 LoRa End-device 구현

✋ LoRa End-device 역할


2번에 해당하는 부분이 End-device입니다. 결국 명령을 게이트웨이로부터 받아서
실제 동작하는 부분을 의미하는건데 본 포스팅에는
아두이노 + 서보모터를 연결 회전력을 이용해서 가전의 스위치를 제어한다고 가정합니다.

✋ LoRa End-device 구성



위에 보이는것처럼 아두이노와 + servo 모터를 연결 거기에 LoRa쉴드를 달아서
LoRa 게이트웨이로 붙어 날라온 msg값을 읽어서 특정 단말 device나
나한테 해당하는 명령어인지를 판단하여서 실행하게끔 구성 계획입니다.

✋ LoRa End-device 구현

이런식의 구성을 통하여 주요 핵심코드는

쪽의 브로드캐스트된 msg값을 열어 자기에 해당하는 쪽을 판단하는 부분입니다.
이 부분을 Id + Msg 의 구성으로 바꾸면 더더욱 많은 단말과, 게이트웨이의 통신을
할수있다는 추가 더 진행해볼만한 연구가 있다고 생각한 부분입니다.

보는것처럼 게이트웨이에서의 명령 하달시 0 ~ 120, 120 ~ 0도의 이동으로 초기
유투브영상처럼 각도만 맞추고 붙여놓은다면 가전의 스위치 누르는 힘정도는 나와서
원격으로 제어가 가능했습니다 !

👉 AI Speaker 구현

✋ AI Speaker 구성



위에 보이는것은 실제 터미널 동작 스크린샷입니다.
python 3버전을 이용 노트북에서 테스트 동작한 사진이구요
구글 어시스턴트 STT sample코드를 이용하여서 구현하였습니다.

✋ AI Speaker 구현

코드부분은 따로 개인 Github에 구글 STT 샘플코드의 라이센스 확인여부후에
기재하겠습니다 !

개략적으로는 우선 D.I.Y에 초점을 둔 AI Speaker인지라 본 포스팅 당시에는
음성으로 ‘예 알겠습니다’ 와 같은 소리는 내어지지 않지만
N-Gram에서의 2~3수준의 바이, 트라이그램 알고리즘을 간략하게 커스터마이징하여
자기가 정해놓은 수준의 위치 또는 단어를 이용하여서 매핑하는 식으로 구현했습니다.

이 부분은 따로 포스팅을 하던지, 개인 깃허브에 Readme에 기재하던지 하겠습니다 ~
아니면 구글 STT example 코드를 가지고 따로 해보셔도 무방할거같습니다

결과가 나오는 부분에 Wemos D1보드가 연결된 URL/트리거건 메서드이름
이렇게 연결하면 외부에서도 접속가능한 컨셉은 유지가 되어지니까요 !

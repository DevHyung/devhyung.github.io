[LoRa]_01_AI스피커를 이용한 LoRa모듈제어

💡 Intro

안녕하세요 "AI 스피커를 이용한 LoRa모듈제어" 프로젝트의 개발진행기를 쓰게 된
경희대 컴퓨터공학과 재학중인 박형준입니다 🙇🏻

우연히 “LoRa” 라는 단어를 접하고 해보고 싶어서 무작정 졸업논문 주제로 잡아서
아두이노를 써본 적도 없었고, LoRa Network의 Framework도 모르고 뛰어들어
헤매고 삽질을 하던 와중에.. 코드주 장병남 대표님의 서울IoT센터에서의
“LoRa야 놀자” 라는 세미나를 듣은 인연으로 남아서 질문도 하고, 메일로도 귀찮게 하면서
제가 일방적으로 도움을 많이 받아가면서 결국에 1달 만에 프로젝트를 완성했습니다.

이에 대해서 고마움도 전해드리고 싶었고, 글을 읽으시는 분들 중 저처럼 처음 아두이노를 입문한 분들은 많이 없으시겠지만..😭 아두이노 뿐만이 아닌 다른 플랫폼을 붙이시거나,
이제 처음 시작해보시거나, 프로젝트를 고민하시는 분들을 위해 이런 개발기를 올리면
“이런것두 가능하구나 !”, “어렵지 않네 ?”를 느낄꺼 같다 라는 대표님의 제안으로
부끄럽지만 개발기를 작성을 하게되었습니다.

말도 잘 못 하는 공대생이고,
SW개발만 하였던 컴공생이고,
맞춤법도 자주 틀리면서..,
이런 개발기도 블로그도 처음입니다😭
서툴어도 부디 넓으신 마음으로 이쁘게 봐주시면 감사하겠습니다 !… 👏

💡 Project Intro

👉 프로젝트 소개

위에서 "AI 스피커를 이용한 LoRa모듈제어" 라고 간단하게 주제를 설명해드렸는데,
사실 제가 연구를 진행한 주제 정확한 이름은
"LoRa IoT 환경망 구축 및 AI 스피커를 이용한 LoRa 모듈 제어" 입니다.
이에대해 부가 설명을 드리자면

LoRaWAN network architecture is deployed in a star-of-stars topology in which gateways relay messages between end-devices and a central network server. The gateways are connected to the network server via standard IP connections and act as a transparent bridge, simply converting RF packets to IP packets and vice versa. The wireless communication takes advantage of the Long Range characteristics of the LoRa physical layer, allowing a single-hop link between the end-device and one or many gateways.

위 그림은 LoRaWAN에 대해 LoRa Alliance™이라는 회사가 정한 네트워크 구조와
그에 대한 설명문입니다.
요약하자면, 게이트웨이와 노드간은 start topology 의 네트워크망을 구성하며
게이트웨이 -> 노드 신호전달과 반대의 경우인 노드 -> 게이트웨이 모두 일어나서
오류체크를 하는것을 권장한다는 내용입니다.

  • 위의 LoRaWAN의 네트워크 구조를 지켜가며 LoRa 통신을 할 수 있는 환경망 구축
  • 구축한 LoRa망을 AI스피커로 제어하고
  • 실제 Node에서 어떠한 동작이 일어나는 ( 여기에서는 스위치를 끄는 예시)

LoRa망구성과, AI스피커, 실제 아두이노에 LoRa모듈을 탑재하여 스위치를 끄는
망구성부터 동작을 내리는 제어부분까지 모두를 제작하는 프로젝트입니다.

이러한 무선(wifi나 블루투스)을 이용해서 아두이노에 명령을 내려
아두이노와 서보모터의 조합으로 스위치를 제어하는 실험은 많았으나
본 프로젝트는 무선의 통신에서는 LoRa모듈을 사용하며,
앱이나 특정행동으로의 명령을 AI 스피커로 간단하게나마 구현했다는 점입니다.

제가 설명을 잘 못해서.. 혹시 이해가 안되셨다면
아래의 컨셉영상과, 실제 시연영상을 보시면 이해하시는데 도움이 되실거같습니다.

👉 프로젝트 목적 및 목표기술

이걸 만들려는 가장 큰 목적은 !
4차 산업의 중요 키워드 중 IoT와 AI 기술을 체험해보고
기존의 IoT제품이 너무 비싸요… 그래서 IoT제품이 아닌걸 IoT 제품으로 싸게
만들어보고 싶다는 바람직한 생각이였습니다.
(사실..제가 복층에 사는데 불끄러 내려가는게 너무 귀찮아서 였다는...읍.으ㅡㅇㅂ.읍..)

그래서 목표기술은
이번 프로젝트에는 스위치를 눌러서 불을 꺼주고 켜주는 것으로 한정해서

LoRa 환경망을 제작,
외부(Web) 와 내부(AI Speaker)에서 제어가능한,
스위치를 자동으로 눌러주는 Kit
(저는 Iot Maker Kit 해서 IMK 라 불렀어요…)를 목표합니다.
